# 📊 Портфель ИИС-3 — Streamlit Dashboard

Дашборд для визуализации брокерских отчётов Сбербанк с автоматическим парсингом HTML.

Подробное описание архитектуры, потоков данных, интеграций, моделей и точек расширения:

## Быстрый старт

```bash
# Установка
cd broker-dashboard
pip install -r requirements.txt

# Импорт первого отчёта
python import_report.py /path/to/S04GWXJ_20042026.HTML

# Запуск дашборда
streamlit run app.py
```

## Структура

```
broker-dashboard/
├── app.py              # Streamlit дашборд (основной)
├── parser.py           # Парсер HTML-отчётов Сбера
├── db.py               # SQLite хранилище
├── import_report.py    # CLI: массовый импорт отчётов
├── fetch_gmail.py      # Автоматическое получение из Gmail (IMAP)
├── requirements.txt
├── .env.example        # Шаблон конфигурации Gmail
├── portfolio.db        # SQLite база (создаётся автоматически)
└── reports/            # Папка для HTML-отчётов
```

## Возможности

- **Обзор**: общая стоимость, структура по типам (pie + bar chart), лидеры роста/падения
- **Позиции**: полная таблица с фильтрацией по типу и сортировкой
- **Пополнения и вычет**: напоминание сколько нужно внести для макс. вычета 52 000 ₽
- **Календарь купонов**: ручное добавление предстоящих выплат
- **Сделки**: все сделки за период
- **История портфеля**: график динамики (после импорта нескольких отчётов)

## Автоматизация (Gmail → парсинг → база)

1. В **mail.ru** настрой пересылку писем от брокера на Gmail
2. Создай [пароль приложения](https://myaccount.google.com/apppasswords) в Google
3. Скопируй `.env.example` → `.env` и заполни
4. Добавь в crontab:

```bash
# Каждый будний день в 10:00
0 10 * * 1-5 cd ~/broker-dashboard && python fetch_gmail.py && streamlit cache clear
```
